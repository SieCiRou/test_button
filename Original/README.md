# VB.NET 按鈕變色反應時間效能分析工具

這是一個基於 Python 的單機自動化測試工具，專門用來量測 Visual Basic.NET (WinForms / WPF) 桌面應用程式中，按鈕在**被點擊後**到**畫面實際完成變色**之間的毫秒級（ms）反應時間。且附有可輸出完整的統計分析報告。

```
## 程式碼架構

本專案由四個核心腳本組成，其邏輯流程如下：

[環境準備]
  |
  +-- detection.py (視窗標題偵測器)
  |     └─ 確認目標程式的精確標題與控制項控制碼。
  |
[執行測試]
  |
  +-- main.py (主測試執行器)
  |     └─ 模擬滑鼠點擊 -> 迴圈掃描像素變色 -> 記錄時間 -> 輸出 Excel。
  |
[進階偵錯]
  |
  +-- test_windows.py (視窗定位測試)
  |     └─ 驗證特定視窗座標與顏色抓取是否正確。
  |
[數據分析]
  |
  +-- performance_analyzer.py (效能分析產生器)
        └─ 讀取 Excel 數據 -> 執行 T-檢定統計 -> 生成 HTML 互動式儀表板。

---
```

```
## 檔案功能說明

1. detection.py
   - 目的：列出目前系統中所有可見視窗。
   - 用途：當無法正確連線到目標程式時，用此腳本確認視窗的正確標題名稱。

2. main.py
   - 目的：自動化測試核心。
   - 功能：
     - 使用 pywinauto 進行點擊動作。
     - 使用 win32gui.GetPixel 進行高頻率顏色輪詢 (Polling)。
     - 自動記錄反應時間並輸出 Excel 報告。

3. test_windows.py
   - 目的：開發與連線測試。
   - 功能：針對單一視窗進行重複次數的顏色變化量測，並計算初步統計值。

4. performance_analyzer.py
   - 目的：將測試結果視覺化。
   - 功能：對比兩組數據（如優化前後），提供平均值、標準差以及 P-Value 統計分析。

---
```

## 特色功能

* **低延遲輪詢** ：捨棄傳統螢幕截圖，改用 `win32gui.GetPixel` API，單次像素偵測時間 < 1ms。
* **物理級模擬** ：透過 `click_input()` 觸發真實滑鼠事件，反映最貼近使用者的真實操作感。
* **科學化分析** ：內建 T-檢定（T-Test）分析，排除硬體隨機抖動產生的虛假數據。

## 安裝與設定

### 1. 環境需求

* Python 3.8 或以上版本
* Windows 作業系統

### 2. 安裝依賴庫

**Bash**

```
pip install pywinauto pywin32 numpy pandas scipy plotly openpyx
```


## 使用步驟

### 步驟 A：定位目標

執行 `python detection.py`，從清單中複製受測程式的完整標題。

### 步驟 B：執行效能測試

1. 開啟 `main.py`，修改 `target_title` 為步驟 A 獲得的標題。
2. 執行腳本：`python main.py`。
3. 測試完成後，會產出如 `RF_Results.xlsx` 的原始數據。

### 步驟 C：產出分析報告

執行分析器：

**Bash**

```
python performance_analyzer.py
```

依據提示輸入兩份 Excel 路徑（例如：before_opt.xlsx 與 after_opt.xlsx），程式將生成 `interactive_performance_dashboard.html` 儀表板。

---

## 輸出數據說明

測試報告與儀表板將包含以下指標：

| **指標名稱** | **說明**                                    |
| ------------------ | ------------------------------------------------- |
| Average (ms)       | 該批次測試的平均反應時間。                        |
| Standard Dev       | 穩定度指標。數值越小，代表反應速度越穩定。        |
| P-Value            | 統計顯著性。若 < 0.05，代表效能優化具備實質意義。 |

---

## 注意事項與故障排除

* **DPI 縮放影響** ：若 Windows 設有縮放（如 150%），可能導致像素點位偏離。建議在 100% 縮放環境下執行。
* **視窗遮擋** ：測試期間，受測程式必須保持在前景且未被其他視窗遮擋。
* **顏色判定** ：若按鈕變色不明顯，請微調 `main.py` 中的 RGB 判定門檻值（Threshold）。
