
# VB.NET 按鈕變色反應時間量測工具 (UI Color Change Latency Timer)

這是一個基於 Python 的單機自動化測試工具，專門用來量測 Visual Basic.NET (WinForms / WPF) 桌面應用程式中，按鈕在**被點擊後**到**畫面實際完成變色**之間的毫秒級（ms）反應時間。本工具支援連續重複測試 100 次，並自動輸出完整的統計分析報告。

## 特色功能

- **高精度計時**：採用 Python `time.perf_counter()`，量測精度達微秒級。
- **超高速像素輪詢**：捨棄傳統耗時的螢幕截圖，改用 Windows 底層 GDI API (`win32gui.GetPixel`)，單次像素偵測時間 < 1ms，確保量測結果不受自動化工具自身效能干擾。
- **真實行為模擬**：使用 `pywinauto` 的 `click_input()` 模擬真實硬體滑鼠點擊，數據最貼近真實使用者體驗。
- **自動化統計**：測試完成後自動計算**平均值**、**最大/最小值**以及**標準差**，便於評估 UI 效能與穩定度。

## 系統需求與套件安裝

本工具適用於 Windows 作業系統環境。請在終端機執行以下指令安裝必要的 Python 套件：

```bash
pip install pywinauto pywin32 numpy
```

### 依賴套件說明：

- **`pywinauto`**：用於尋找 Windows 視窗控制項並模擬滑鼠點擊。
- **`pywin32`**：呼叫 Windows 原生 API 進行高效能像素讀取。
- **`numpy`**：用於計算 100 次測試數據的平均值與標準差。

## 使用步驟

### 步驟 1：準備您的 VB.NET 受測程式

請確保您的 VB.NET 應用程式滿足以下條件：

1. 主視窗標題（Title）為 `"Form1"`（或與腳本中的設定一致）。
2. 目標按鈕顯示文字為 `"點我測試"`（或是擁有不重複的 `auto_id`）。
3. **重要**：按鈕在變色後，需要有機制能夠恢復原色，以便進行下一次迴圈測試（例如：點擊後延遲 50ms 自動變回原色，或由 Python 腳本觸發重設按鈕）。

### 步驟 2：配置 Python 測試腳本

開啟 `color_timer_test.py`，根據實際狀況調整以下參數：

```python
# 更改為您 VB.NET 視窗的實際標題
app = Application(backend="win32").connect(title="Form1") 

# 更改為您目標按鈕的辨識特徵 (可以使用 title, auto_id 等)
btn = main_win.child_window(title="點我測試", control_type="Button") 
```

### 步驟 3：執行測試

1. 先手動啟動您的 VB.NET 應用程式，並讓其保持在畫面上（不要最小化）。
2. 執行 Python 腳本：

```bash
python color_timer_test.py
```

## ⚠️ 注意事項與故障排除

1. **找不到視窗或控制項？**
   若畫面提示找不到控制項，請在點擊前加入 `main_win.print_control_identifiers()` 來列印出視窗內所有控制項的階層與名稱，並以此修正 `child_window()` 的篩選條件。
2. **測試中滑鼠被控制？**
   因為使用的是 `click_input()`（物理模擬），測試進行時**請勿手動移動滑鼠或操作其他視窗**，以免干擾點擊準確度。
3. **畫面縮放（DPI）影響座標？**
   如果您的 Windows 有開啟文字放大（例如 125% 或 150%），可能會導致座標抓取偏移。建議在 Windows 顯示設定中將縮放調整為 100% 進行測試，以獲得最精準的結果。
